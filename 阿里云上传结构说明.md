## 阿里云 OSS 上传结构说明

**目标**：把 `Test1` 整站按原目录结构上传到 OSS，保证页面里的相对路径（比如 `../All.mp3`、`1.主页/xxx`、`Q0/` 等）都能正确访问。

### 二、用控制台上传（不写命令行）

1. 打开 OSS 对应的 Bucket → **文件管理**。
2. 点击 **上传** → 选择 **上传文件夹**（不同浏览器可能叫法略有差异）。
3. 选择本地的 `Test1` 文件夹：
   - 如果 Bucket 中 **没有其他内容**，可以直接把整个 `Test1` 作为根目录上传；
   - 如果 Bucket 已经有内容，建议：
     - 先在 Bucket 里新建一个目录（如 `h5/`），
     - 再把 `Test1` 里的所有内容（不含最外层 `Test1` 文件夹本身）上传到这个 `h5/` 目录里。
4. 上传完成后，在 OSS 控制台的目录树中，确认 `1.主页/`、`Q0/`、`Q1/`……、`结束/` 都在正确的位置。

### 三、用 ossutil 同步（命令行方式）

假设你已经按照阿里云文档安装并配置好了 `ossutil`：

```bash
# 把本地 Test1 目录整体同步到 Bucket 根目录
ossutil cp -r /Users/norie/Downloads/Test1/ oss://你的Bucket名/ --update
```

- `-r`：递归拷贝子目录。
- `--update`：只上传新增或更新的文件，后续改动时重复执行即可增量同步。

如果想把站点放在 Bucket 下的子目录（比如 `survey/`），可以改成：

```bash
ossutil cp -r /Users/norie/Downloads/Test1/ oss://你的Bucket名/survey/ --update
```

然后在 OSS 的「静态页面」里，把默认首页设置为：

- 若上传到根目录：`1.主页/intro_public.html`
- 若上传到 `survey/` 目录：`survey/1.主页/intro_public.html`

这样就可以保证 **Test1 整站按原目录结构上传**，手机访问时所有页面和 BGM 路径都会正常工作。
