# Hello World 内测 · H5 互动问卷 — 项目工作回顾

> 给 CEO 汇报用：整体思路、创作流程、产品形态、技术实现与交付物。

---

## 一、创作与生产流程（方法论）

整体思路：**先定题与互动，再定美术与调性，最后用 Figma + AI + 代码迭代实现，音频收尾**。可复用到后续 H5 互动项目。

### 阶段一：题目与互动设计

1. **选定问卷问题**  
   明确要收集什么（如：第几次参加、第一印象、最烦的设计、节奏感知、给 Peter 的信），并排好顺序与跳转逻辑。

2. **为每题设计趣味化互动**  
   每个问题对应一种「玩法」，而不是单纯点选项：  
   - 例如「第几次」→ 注射瞄准（选血管 A/B）、「第一印象」→ 黑胶选歌、「谐音/听写」→ 听音频填四字、「节奏」→ 背包选项 + 小游戏、「写信」→ 文本框 + 春姬 AI 润色。  
   这样既收数据，又强化记忆点和传播性。

3. **由互动反推美术与整体调性**  
   根据上述玩法，统一世界观与视觉语言：  
   - 本项目中定为「赛博 / 旧时代遗产」梗 + 春姬（Niuma）AI 人设；  
   - 配色、字体、控件风格（扫描线、霓虹边框、终端感）都围绕这一套调性展开，保证每一 part 都像同一世界里的不同关卡。

### 阶段二：视觉与交互实现（Figma + AI + Vibe Coding）

4. **用 Figma 打通 Claude，产出主题调性与美术模版**  
   - 在 Figma 中描述需求（问卷主题、调性、参考风格），让 AI 协助产出：主色、字体、组件风格、典型页面框架等，形成可复用的「美术模版」。  
   - 这样先锁定「长什么样」，再往下做交互和代码，避免反复大改。

5. **按 part 定文案与对话，在 Figma 里落稿**  
   - 为每个 part（主页、Q0～Q4、结束页）写好：开场白、题目说明、按钮与过渡文案、春姬吐槽/彩蛋等。  
   - 把文案和实际需求同步进 Figma（标注在稿子上或写在说明里），便于前后端/协作对齐「最终要呈现的字和节奏」。

6. **根据实际交互形式做代码与 UI 调整（Vibe Coding）**  
   - 在已有美术模版和文案基础上，用 AI 辅助写/改代码（HTML/CSS/JS），实现真实交互。  
   - 实现过程中根据「真实点击、滑动、反馈」再微调 UI（布局、动效、节奏），形成「设计稿 → 代码 → 体验反馈 → 再调设计/代码」的短循环，而不是一次性交付。

### 阶段三：音频收尾

7. **在 AI 音乐网站为每个页面选 BGM**  
   - 按每个 part 的情绪与节奏选曲：如主页偏仪式感、Q1 黑胶偏复古、Q3 节奏关偏动感、结束页偏安静回顾。  
   - 统一音量与是否循环，保证整条线听感连贯。

8. **用 AI 配音完成趣味音频**  
   - 需要人声的地方（如春姬提示、彩蛋台词）用 AI 配音，风格与人设一致，增强「角色感」和传播点。  
   - 与 BGM 一起嵌入页面，并处理好自动播放策略（如首次点击/触摸再播），兼顾体验与策略限制。

## 二、项目目标与形态

**目标**：做一套可公网访问的「内测问卷」H5，用于收集用户反馈并强化品牌记忆，而不是传统表单填空。

**形态**：
- **单线流程**：主页 → Q0（自我归序）→ Q1～Q4（四类互动检测）→ 结束页（档案回顾 + 人格标签）
- **统一世界观**：赛博/旧时代梗 + 春姬（Niuma）AI 助手人设，贯穿开场白、题目说明与彩蛋
- **强互动**：每道题都是小玩法（注射瞄准、黑胶选歌、谐音听写、节奏/背包、写信），答案存本地，结束页统一回顾

## 三、整体工作思路（落地细节）

### 1. 流程与关卡设计

- **主页**：落地页/启动页，完成后自动跳转 Q0。
- **Q0（自我归序）**：  
  - 概念上做成「进门先报身份」的轻量身份环节，文案升格为「通名」「自我归序」等有内涵的短句，而不是直白的「身份检验」。  
  - 交互：初诊/复诊二选一 + 针筒推动注射（缓慢推动 + 俏皮/双关文案），结果写 `survey_q0`，过渡文案衔接「正式测评」。
- **Q1～Q4**：沿用并统一既有互动形式（黑胶、谐音、节奏、写信），进度条统一为 **5 步（1/5～5/5）**，BGM 与过渡逻辑与 Q0 对齐。
- **结束页**：读取 `survey_q0`～`survey_q4`，展示「测评档案」与人格标签，支持分享/复制链接。

### 2. 体验与文案

- **开场白**：Q0 采用「自我归序——旧时代人类最温柔的群体默契」等短句，与 Q1～Q4 的「旧时代遗产」风格一致。
- **过渡**：Q0 结束后不写「样本注入/核对档案」，改为「自我归序已记录 · 即将进入正式测评」等衔接语，带一点沙雕/反转感（如「准备好迎接拷打」），过渡后快速跳转，不长时间停留。
- **彩蛋**：去掉 B 选项关灯彩蛋；B 保留「老熟人」专属过渡文案；针筒推动过程有分阶段俏皮文案，增强记忆点。

### 3. 技术实现

- **纯静态**：全站 HTML + CSS + JS，无后端；答案用 `localStorage`（`survey_q0`～`survey_q4`）存储，结束页统一读取展示。
- **兼容与性能**：  
  - 各页 BGM 统一逻辑：加载尝试播 + 首次点击/触摸兜底（应对浏览器自动播限制）。  
  - 大体积音频：原始 .wav 不进 Git；用脚本裁切为「最多 120 秒」并转 128k MP3 后提交，保证推送不超时、线上可播。
- **部署**：Git 初始化 + `.gitignore` / `.vercelignore` 控制体积；根目录 `index.html` + `vercel.json` 做入口与重写；推送到 GitHub（norie7k/H5Project）后由 Vercel 一键部署公网。

### 4. 遇到的问题与解决

| 问题 | 处理方式 |
|------|----------|
| 推送 225MB 时 HTTP 408 超时 | 将 8 个大 .wav 排除出 Git，用脚本生成 120 秒 MP3 再提交，体积降至可推送范围 |
| Q0 与 Q1～Q4 的进度/衔接感不统一 | 进度统一为 5 步；Q0 过渡文案改为「正式测评」衔接，并缩短停留时间 |
| 结束页缺少 Q0 结果 | 结束页增加对 `survey_q0` 的读取与展示（自我归序 · 初诊/复诊） |

## 五、问卷故事内容总结

整条问卷用**同一套世界观**讲了一个完整小故事：

- **设定**：背景是「第六纪元 · 殖装驾驶员检测系统」——你作为候选人，在 AI 助手 **春姬（Niuma）** 的引导下，完成一轮「旧时代人类遗产」的趣味检测，本质是在做内测反馈收集。

- **主页**：从**身体组装**开始——把躯干、四肢等拖进格子完成殖装同步，输入姓名，进入检测流程。强调「殖装兼容性」「救世军」等赛博味，同时用 BGM 和仪式感把气氛拉满。

- **Q0（自我归序）**：春姬先让你「通名」——用**针筒注射**选「初诊 / 复诊」，对应「第一次来 / 来过几次」。文案强调这是旧时代人类「最温柔的群体默契」：不靠数据标定，只凭自知，选一条属于自己的轨迹。完成后衔接「即将进入正式测评」。

- **Q1（黑胶唱片）**：把题目包装成**旧时代听觉考古**——那时「没毒气、没殖装、没魔方」，人类靠黑胶听「艺术」。你要在时光纹路里选歌，实则是**第一印象 / 体验评价**的趣味化。

- **Q2（谐音梗与抽象话）**：题目变成**旧时代互联网遗产**——人类用「互联网」培养皿沟通，爱把字拆、缩、扭成团，美其名曰「表达艺术」。你参加**听力大考**，听写解码，对应的是**对设计/体验的听写与选择**。

- **Q3（指尖蹦迪术）**：包装成旧时代**赛博运动**——十指练成残影、「抽搐」够快灵魂就能逃离肉身。实际是**背包/节奏困惑度**的选项与小游戏，春姬会吐槽或给过关评语。

- **Q4（纸笔陈情帖）**：旧时代人类**最真诚的谏言方式**——握笔伏案、一字一句落在纸上。你在这里**给 Peter 写信**，写下真实想法；春姬可帮你「升华」成 AI 润色版，但强调 Peter 更爱看真实想法。

- **结束页**：春姬**TOKEN 用尽**，进入「FINAL TRANSMISSION」——用最后算力把你的**测评档案**（Q0～Q4 答案）和**给 Peter 的信**打包传走，并根据答案算出一个**人格标签**（如「资深差评师」「野生策划」「整活大师」「神秘玩家」等），附带一句 Peter 或系统的吐槽。页面标题是「春姬 · TOKEN LIMIT REACHED」，分享语里写「春姬用最后的 token 帮我整理了报告」，把整条问卷收束成「春姬帮你跑完检测并传话给制作人」的结局。

**一句话**：在「第六纪元」的殖装检测里，被春姬带着用「旧时代遗产」的方式完成五关趣味答题，最后春姬用尽 token 给玩家生成一份检测报告。

### 选项操作与彩蛋

- **Q0**：选项 A 针筒会抖；选项 B 针头变大；两个选项推动过程中按进度切春姬俏皮文案（如「慢慢来…」「快到底了…」）。
- **Q1**：选歌后点「确定」触发三次确认，点「取消」会随机出梗；选择选项 E 触发歌词对应的关灯黑屏彩蛋。
- **Q2**：建议解锁所有选项的听力答案填写。
- **Q3**：建议把所有选项都体验一遍。
- **Q4**：一定体验「一键 AI 润色」功能。

## 六、一句话总结

**把内测问卷做成一条线的 H5 互动流程**：用「自我归序 + 四轮正式测评」的结构、统一春姬人设与赛博风文案、纯静态 + localStorage + Vercel 部署，在保证可公网访问和完整 BGM 的前提下，解决了大音频导致的推送超时，并统一了进度与过渡体验，便于对内汇报和后续迭代。
